Cette proposition de projet PSTL se situe dans le cadre d’un effort plus général d’implantation d’un modèle de composants minimal pour Java. Ce modèle est actuellement implanté et utilisé dans le cadre de projets de recherche où il a permis de construire des applications à grande échelle (une dizaine de milliers de composants s’exécutant sur des dizaines de machines virtuelles Java). Il est également utilisé par les étudiants de l’UE de M2 ALASCA pour réaliser le projet de semestre.

Java et le modèle à composants BCM offrent de nombreuses possibilités de modifier dynamiquement le code d’un composant et ses connexions avec les autres composants. Cependant, le modèle BCM est multi-tâches par essence puisque de nombreux composants ont leurs propres threads et les appels de méthodes à distance asynchrones favorisent le parallélisme d’exécution. Se pose alors la difficulté de l’indisponibilité d’un composant pendant sa modificaiton alors que les autres composants peuvent continuer à l’appeler. L’objectif du projet est concrétiser la capacité de modification dynamique des composants dans le modèle par une gestion explicite de l’arrêt puis du redémarrage des threads pendant les modifications (changement des connexions avec d’autres composants, modification du code, etc.).
